> Tags: Zero Cost AI, Slow Thinking Systems

Recently have been tinkering with small language models. These models are able to Run on my Mac book, uses around 7-30GB Unified Memory, and able to do lot of things, are good at tool calling, can accomplish trivial tasks like calling the right API as long as the task is well defined. (my reference model in this case is Gemma4 e4B, Gemma4 27B)

I actually call them Zero Cost Intelligence (well exactly not free, but almost negligible contrast to API cost of LLMs). Well intelligence might be overrated term so i prefer it to be called as Language and Reasoning Simulators. And that made me wonder, is it possible to build a system that uses group of these models (small models running on consumer hardware, or not needing expensive GPUs) and able to do some tasks (subset of enterprise level tasks)

Plus, not all tasks need realtime output. For e.g: when you are sleeping, and these LLM agents working in the background, it is not necessary that output token speed should be realtime. That is why for e.g: if you have a workflow or tasks that doesn't need to provide immediate output, then why not run them using system of these small models.

System of Small Models: As we know small LLM are not very smart, so in order to improve reliability of tasks, we need to build up a system that is able to channelize the model towards right direction and eventually towards right output.

Example System could look like:

- A Large Reasoning Model create a trajectory plan.. very detailed steps for the other less intelligent model to follow.
- A small langauge model then follows that trajectory.
- There could be then another agent using small model that verified at the end...
